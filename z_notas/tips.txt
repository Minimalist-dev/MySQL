- Columnas diplicadas: 
    ej: si existe la fecha de nacimiento (NO CREAR LA COLUMNA EDAD)
    ej: si existe la columna en una tabla (EVITAR CREARLA EN OTRA)
    ej: (CREAR TABLA SEPARADA) si la información es parte de 2 o mas tablas
    ej: si la tabla contiene información que al consultarla no es necesaria (POSIBLEMENTE SERÍA MEJOR SEPARARLA)

- Triggers:
    ej: proteger la base de datos de (ALTERACIONES)
    ej: (REGISTROS) de cambios
    ej: (COPIAS) de seguridad
    ej: ejecutar 2 o mas sentencias a la vez, en vez de transacciones

- Funciones:
    ej: si no existe el front-end
    ej: algoritmos en el servidor

- Vistas: 
    ej: si no existe el front-end
    ej: crear bases de datos personalizadas

- Procedimientos almacenados:
    ej: mejorar el rendimiento de las (sentencias ejecutadas)
    ej: cuando se quiere restrinjir el acceso (SEGURIDAD) desde el servidor 
    ej: cuando se quiera manipular los datos desde el sevidor

- Transaciones:
    ej: cuando se quiere ejecutar (2 o MAS SENTENCIAS)
    ej: Actualizar, insertar o eliminar registros en la base de datos
    ej: Alternativa a los triggers, de manera mas detallada desde la interfaz. Solo si los triggers no bastan.

- Eventos: 
    ej: cuando se quieran realizar tareas automaticas
